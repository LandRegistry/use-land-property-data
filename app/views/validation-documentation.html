Add validation to your prototype

There are various types of validation you can do - for example, to ensure the field isn't blank, if a checkbox or radio button has been selected, if what's entered is long enough, if it's a pattern like a postcode.

If you want to constrain or validate how much text can be inputted - there are also other ways to do this, such as setting a field length attribute (minlength or maxlength), or using the character count govuk code for the text area.

The validation uses the javascript library validate.js, which contains all these standard validators and the ability to create custom validation rules. The documentation can be hard to read - to translate into how to use in this kit, so we've included common examples.
