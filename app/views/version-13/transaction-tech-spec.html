Overview

Transaction Data includes 5 different datasets that show how many customer applications we completed during the previous month. The applications are:

  • first registrations
  • leases
  • transfers of part
  • dealings
  • official copies
  • official searches

Specification scope

Inclusions and exclusions

The data includes most applications but excludes:

  • bankruptcy applications
  • bulk applications
  • applications that we've received but not yet completed

Data quality

Transaction datasets are created automatically from HM Land Registry databases. We try to make sure that the data is accurate. This cannot be guaranteed and the data may contain errors.

Data capture information

This dataset contains data we collect about customers and their applications. Most applications from professional customers are made electronically. They send applications either via the Business Gateway or directly from the lender’s computer system. The applications are entered automatically to the day list.

Data maintenance

The datasets are updated on a monthly basis on the 15th working day of that month.

Data product delivery

Access to the dataset

There are 5 datasets which show the data in different ways. You can choose which dataset best suits your needs.

File structure

The data in each file will:

  • use a comma to separate each field: ‘,’ (ASCII 44)
  • enclose all fields within double-quotes: ”” (ASCII 34)
  • send blank fields without a character between double quotes: ””
  • format dates as DD-MM-YYYY HH:MM:SS
  • use UTF-8 and ASCII encoding (the way characters are coded for websites)
  • remove line separation the data
  • have the column heading name as the first row

File name

The file will be structured as follows:

Transaction-<1>-<2>.csv

<1> Year

<2> Month

Example file name structure: Number-of-applications-by-region-2017-10.csv
