Overview

Transaction data is a dataset that shows how many customer applications HM Land Registry (HMLR) completed in the preceding month. We publish 5 files each month based on customer and location.

Specification scope

Inclusions and exclusions

The data includes the following application services:

  • first registrations
  • leases
  • transfers of part
  • dealings
  • official copies
  • official searches

Pending applications, bankruptcy applications and bulk applications are excluded.

Relationship to other HM Land Registry data products

The Account Customer field may allow relational links to be made with other HMLR datasets that also contain this field.

Description of the data fields in the applications by authority file
Field name Data type Mandatory Description
Local Authority CharacterString Yes The local authority under which the transaction took place
FR CharacterString No An application for a first registration of land both freehold and leasehold. For leasehold this applies when the landlord’s title is not registered
DFL CharacterString No An application for the registration of a new lease granted by the proprietor of registered land

DLG CharacterString No An application in respect of registered land. This includes transfers of title, charges and notices

OC1 CharacterString No An application to obtain an official copy of a register or title plan represents a true record of entries in the register and extent of the registered title at a specific date and time. OC2 CharacterString No An application to obtain a copy of a document referred to in the register or relates to an application. This includes correspondence, surveys, application forms and emails relating to applications that are pending, cancelled or completed
Total CharacterString Yes Total applications

Data quality

We try to make sure that our public data is accurate, but we cannot guarantee that it is free from errors or fit for your purpose or use.

Data capture information

This dataset contains data we collect about customers and their applications. Most applications from professional customers are made electronically. They send applications to HMLR either via the Business Gateway or directly from the lender’s computer system.

Data maintenance

We update the dataset monthly. It’s available to access on the 15th working day of the following month.

Data product delivery

Access to the dataset

The dataset is available as CSV. files and is released under the Open Government Licence (OGL). Transaction data is also available in a machine-readable format as linked data.

File structure

The data is provided as csv files. The data in each file will:

  • use a comma to separate each field: ‘,’ (ASCII 44)
  • enclose all fields within double-quotes: ”” (ASCII 34)
  • send blank fields without a character between double quotes: ””
  • format dates as DD-MM-YYYY HH:MM:SS
  • use UTF-8 encoding (the way characters are coded for websites)
  • have line separation removed
  • have the column heading name as the first row

File name

The file names will be structured as follows:

<Transaction>-<1>-<2>.csv
<1> Year <2> Month
Example file name structure: Number-of-applications-by-region-2017-10.csv
